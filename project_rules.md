# 卡包助手项目 AI 编程规范

## 项目概述

本项目是一个微信小程序，用于管理信用卡和分期账单。项目采用微信小程序原生开发框架，包含云函数后端服务。

## 1. 项目结构规范

### 1.1 目录结构
```
卡包助手/
├── miniprogram/           # 小程序前端代码
│   ├── pages/            # 页面文件
│   ├── utils/            # 工具类
│   ├── images/           # 图片资源
│   └── custom-tab-bar/   # 自定义tabbar
├── cloudfunctions/       # 云函数
├── prototype/           # 原型文件
└── project.config.json  # 项目配置
```

### 1.2 文件命名规范
- 页面文件：使用小写字母和连字符，如 `settings.wxml`
- 工具类：使用 PascalCase，如 `CardDataManager.js`
- 图片文件：使用小写字母和连字符，如 `card-active.svg`

## 2. 代码编写规范

### 2.1 微信小程序特定规范

#### WXML 规范
- 使用语义化的 class 名称
- 遵循 BEM 命名规范：`.block__element--modifier`
- 避免内联样式，统一使用 WXSS
- 正确使用小程序组件和 API

#### WXSS 规范
- 使用 rpx 作为尺寸单位（1rpx = 0.5px 在 iPhone6 上）
- 遵循移动端适配原则
- 使用 CSS3 特性时考虑兼容性
- 避免使用 `!important`

#### JavaScript 规范
- 使用 ES6+ 语法
- 正确处理异步操作（Promise/async-await）
- 遵循小程序生命周期
- 使用 `wx.` API 前缀调用小程序 API

### 2.2 样式开发规范

#### 尺寸转换
- 设计稿 px 转 rpx：`rpx = px * 2`（基于 iPhone6 750px 宽度）
- 常用转换：16px = 32rpx, 24px = 48rpx

#### 布局规范
- 优先使用 Flexbox 布局
- 合理使用 `display: grid` 进行网格布局
- 统一边距和内边距规范

## 3. AI 编程行为规范

### 3.1 代码修改前的必要检查

1. **文件存在性检查**
   - 使用 `view_files` 或 `list_dir` 确认文件存在
   - 检查文件路径的正确性

2. **代码上下文理解**
   - 查看相关文件的完整结构
   - 理解现有代码的设计模式
   - 确认依赖关系和引用

3. **原型对比**
   - 参考 `prototype/` 目录下的 HTML 原型
   - 确保实现与设计稿一致

### 3.2 代码修改规范

1. **渐进式修改**
   - 每次只修改一个功能点
   - 避免大范围重构
   - 保持代码的向后兼容性

2. **样式修改规范**
   - 修改前先理解现有样式结构
   - 保持样式命名的一致性
   - 确保响应式设计不被破坏

3. **功能实现规范**
   - 遵循小程序 API 使用规范
   - 正确处理数据绑定和事件处理
   - 考虑性能优化

### 3.3 错误预防机制

1. **路径处理**
   - 始终使用绝对路径
   - 确认文件路径分隔符正确（Windows: `\`）

2. **代码验证**
   - 检查语法正确性
   - 确认变量和函数名拼写
   - 验证 API 调用的正确性

3. **测试验证**
   - 修改后进行功能测试
   - 检查控制台错误信息
   - 验证视觉效果是否符合预期
